HOW IT WORKS
📸
1. SEE
Screenshot → JPEG → LLava
Takes a screenshot every step using MSS (fastest Python screencap library). Resizes to 50% for LLava speed. LLava describes: "Chrome browser open, Amazon homepage visible, search bar at top."
🧠
2. UNDERSTAND
LLava reads screen context
LLava (vision LLM) reads the screenshot and returns a description of what's visible. This gives JARVIS real-time awareness of the screen state before every action — not guessing.
📋
3. PLAN
Llama3 breaks into steps
Llama3:8b receives: screen description + your instruction → creates a JSON array of 5-25 steps. OR if instruction matches a known Playbook, the pre-built steps are used instantly (faster + more reliable).
🖱️
4. EXECUTE
pyautogui controls PC
Each step runs via pyautogui: open_app, navigate (Ctrl+L), click (LLava finds exact pixel coords), type, press keys, scroll, hotkeys. LLava is called again for every click to find where exactly to click on screen.
✅
5. VERIFY
LLava confirms it worked
After every click/navigate/press: takes new screenshot, LLava checks "did this step succeed?". If NO → self-correction: phi3:mini suggests an alternative approach, JARVIS retries. Emergency stop: move mouse to top-left corner.
