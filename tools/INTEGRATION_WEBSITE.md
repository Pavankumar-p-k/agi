# JARVIS Website Engine — Integration Guide

## What was replaced

The old `tools/website_generator.py` loaded pre-downloaded template repos from disk,
filled placeholders with an LLM, and produced inconsistent results.

The new engine generates every page **from scratch** using a 5-step AI pipeline:

```
Step 1 — Research    search_tool → 5 keyword/insight strings
Step 2 — Design      LLM → :root{} CSS variable block (colours, fonts, spacing)
Step 3 — Generate    LLM → complete HTML5 page per page name
Step 4 — Post-proc   validate HTML, inject CSS, write assets/sitemap/README
Step 5 — Preview     http.server on random free port, returns preview_url
```

---

## File layout

```
tools/website_generator.py    ← REPLACE THIS FILE (drop in the new one)
api/website_routes.py         ← NEW: FastAPI router  (add to app.include_router)
jarvis_website_cli.py         ← CLI helpers to paste into jarvis.py
```

## Integrating the API routes

In your main FastAPI entry point (usually `api/app.py` or similar):

```python
from api.website_routes import router as website_router
app.include_router(website_router)
```

That exposes:

| Method | Path | Description |
|--------|------|-------------|
| POST | `/website/generate` | Start generation job → `{job_id}` |
| GET  | `/website/status/{job_id}` | Poll progress (0-100) + result |
| GET  | `/website/jobs` | List all jobs |
| POST | `/website/preview` | Serve existing site dir |
| POST | `/website/stop` | Kill preview server(s) |
| GET  | `/website/styles` | List style presets |

## Integrating the CLI

In `jarvis.py`, find the `add_subparsers` call and dispatch block, then add:

```python
# 1. Import
from jarvis_website_cli import _add_website_commands, cmd_website

# 2. Register sub-command (near other add_parser calls)
_add_website_commands(sub)

# 3. Dispatch (inside the big if/elif block)
elif args.command == "website":
    return cmd_website(args)
```

---

## CLI usage

```bash
# Build a site
jarvis website build "Coffee Shop" --pages index,about,menu,contact --style warm

# Build with custom output dir
jarvis website build "My Portfolio" --style creative --output ./portfolio_site

# List style presets
jarvis website styles

# Preview an existing site
jarvis website preview ./portfolio_site

# Stop all preview servers
jarvis website stop

# Stop one specific port
jarvis website stop --port 54321
```

## API usage

```bash
# Start generation
curl -X POST http://localhost:8000/website/generate \
  -H "Content-Type: application/json" \
  -d '{"topic":"Coffee Shop","pages":["index","about","menu","contact"],"style":"warm"}'
# → {"job_id": "abc-123", "poll_url": "/website/status/abc-123"}

# Poll until done
curl http://localhost:8000/website/status/abc-123
# → {"status":"running","progress":55, ...}
# → {"status":"done","progress":100,"result":{...}}
```

## Fallback behaviour

If the LLM is completely unavailable, each page is generated by `_fallback_page()`,
a high-quality hand-crafted template (NOT the old placeholder garbage).
Pages generated via fallback are marked `"fallback": true` in the result dict.

LLM lookup order:
1. `core.llm_router.complete` (Jarvis router — tries Anthropic/OpenAI/Gemini)
2. `http://localhost:11434/api/generate` (Ollama local — model: llama3)
3. `_fallback_page()` static template
