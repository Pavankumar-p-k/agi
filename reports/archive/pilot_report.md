## State Leak Observations

The shared session means page state persists between tasks. 19 tasks started on a non-blank page (the URL from a previous task).

### Impact on Results
- Tasks that require navigating to a specific URL are **not** affected — the model navigates before searching.
- Tasks that depend on the current page state (e.g., checking if already on a page) **may** be affected — the shared session could artificially inflate success for tasks that inherit a relevant URL.
- Tasks that are sensitive to page overload (many navigations) see **accumulated** state, which could degrade performance over time.
